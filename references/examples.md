# Agent PRD 短样例

本文件只放短样例片段。需要完整 PRD 时使用 `agent-prd-template.md`，需要工程契约时使用 `engineering-contracts.md`。

## Agent 套件判断片段

当一个办公产品同时处理会议、邮件、日程和文档写入时，应优先定义为 Agent 套件。总入口 Agent 负责识别任务、展示状态、调度场景 Agent 和管理确认；会议 Agent、邮件 Agent、日程 Agent、文档 Agent 分别处理各自场景。涉及发送、共享、创建外部日程等动作时，必须进入用户确认流程。

## 待确认项片段

| 待确认项 | 影响 |
|---|---|
| 是否接入真实邮箱与日历 | 决定工具调用契约和权限策略。 |
| 首版是否允许外部写入 | 决定自主等级、确认流程和审计要求。 |
| 是否需要长期记忆 | 决定数据保留策略和用户控制入口。 |
