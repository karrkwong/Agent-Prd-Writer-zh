# Agent 工程契约参考

本文件只在用户要求“开发可开工”“工程可落地”“可以直接开发”时使用。PM 不需要独立完成全部工程细节，但必须提供足够清楚的产品规则、风险规则和验收口径。

## 1. Agent 运行契约

每个关键 Agent 至少写清：触发事件、输入、输出、可调用工具、工具调用上限、停止条件、需要用户确认的条件、异常降级方式。

| 字段 | 说明 |
|---|---|
| 触发事件 | 用户点击、邮件进入、日程临近、上游 handoff 等 |
| 输入 | 需要哪些信息才能工作，哪些字段允许为空 |
| 输出 | 用户或下游系统拿到什么，是否包含来源和缺失信息 |
| 工具范围 | 可调用工具白名单，区分真实工具、mock、待确认 |
| 停止条件 | 成功、失败、取消、超时、信息不足、待确认 |
| 用户确认 | 哪些结果或动作需要用户确认 |
| 降级方式 | 保留草稿、提示授权、手动上传、转人工等 |

## 2. 输入 / 输出 schema

需要机器可读时再展开 JSON Schema / Pydantic / Zod。PRD 中至少写清字段含义、必填项、允许为空项、来源和风险等级。

最小输入字段：`task_id`、`user_id`、`user_intent`、`source_context`、`constraints`。

最小输出字段：`task_id`、`agent_name`、`status`、`result`、`sources`、`missing_info`、`next_actions`、`risk_level`、`requires_user_confirmation`、`error`。

## 3. 工具调用契约

| 字段 | 要求 |
|---|---|
| 工具状态 | 真实接入 / mock / 待确认 |
| 用途 | 读什么、写什么、生成什么 |
| 输入输出 | 关键参数和返回字段 |
| 权限 | 读取、写入、发送、共享、管理员配置等 |
| 用户确认 | 外部写入、对外发送、共享、购买、删除必须确认 |
| 失败处理 | 权限不足、超时、参数错误、服务不可用、结果为空、部分成功 |
| 重试与回滚 | 哪些失败可重试，执行后能否撤销 |
| 日志 | 记录请求 ID、参数摘要、结果摘要、耗时、错误码；不保存敏感原文 |

## 4. Agent 交接 payload

所有 Agent 交接必须结构化。至少包含：`task_id`、`user_intent`、`source_context`、`current_agent`、`next_agent`、`handoff_reason`、`required_tools`、`risk_level`、`needs_user_confirmation`、`missing_info`、`expected_output`。

来源不足时：

- `risk_level` 至少为 `medium`。
- `needs_user_confirmation` 设为 `true`。
- `missing_info` 写清缺少的来源、权限或用户判断。
- `expected_output` 不得要求下游输出确定性结论，只能要求输出草稿、候选项或待确认问题。

## 5. 状态转移规则

状态机不得只列状态，必须写清进入条件、下一状态、用户可执行动作、系统可执行动作、是否允许取消、是否允许重试、是否会产生外部写入、是否必须记录审计日志。

最低要求：

- 外部写入状态必须记录审计日志。
- 执行中状态不允许静默失败，必须进入已完成或失败。
- 失败状态必须说明可重试和不可重试条件。
- 用户取消后不得继续执行外部动作。

## 6. 记忆与数据策略

区分任务上下文、来源引用、长期偏好和敏感信息。

- 可以保存：用户明确授权的偏好、任务状态、来源引用、确认记录。
- 不能保存：未经授权的邮件正文、会议敏感原文、个人隐私、商业机密、密码、密钥、未确认的长期偏好。
- Demo 默认不做长期记忆；MVP 只保存任务级状态和来源引用；Production 再考虑用户授权下的长期偏好。
- 用户必须能查看、修改、删除长期记忆。

## 7. 日志、审计与可观测性

日志只保存排查、审计和回归需要的信息。

| 日志类型 | 最低字段 |
|---|---|
| 任务日志 | task_id、user_id 或匿名标识、agent_name、状态、触发事件、输入摘要、输出摘要、风险等级、耗时 |
| 工具调用日志 | task_id、tool_name、call_id、参数摘要、结果摘要、是否写入、是否确认、错误码、耗时、重试次数 |
| 用户确认日志 | confirmation_id、task_id、确认动作、预览摘要、最终执行摘要、确认时间、是否修改草稿 |
| 错误日志 | error_id、task_id、agent_name、错误等级、错误码、恢复动作 |

限制：不保存邮件正文、会议原文、完整附件内容、密钥、密码、个人敏感信息。联系人、邮箱、手机号、客户名称、合同金额等敏感字段必须脱敏。

## 8. 工程测试计划

开发前至少定义：单 Agent 单元测试、多 Agent 交接测试、工具调用测试、权限与确认测试、失败恢复测试、安全与隐私测试、回归测试、端到端验收测试。

PRD 只需要写测试目标和通过标准；样本、脚本和自动化方式由工程 / QA 补齐。
