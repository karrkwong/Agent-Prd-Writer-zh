# PRD-writer

`PRD-writer` 是一个用于输出中文 PRD / 需求文档的 Codex skill。

它面向内部产品评审和研发协作场景，适合把产品目标、业务素材、截图、飞书文档或口头描述整理成结构稳定、可评审、可开发跟进的 PRD。

## 适用场景

- 从零撰写中文 PRD。
- 基于已有 PRD 做审视、去重、补充或重写。
- 将线上飞书文档作为基线，按用户最新修改继续收敛内容。
- 输出包含 Agent、场景 Agent、工具调用、用户确认、异常兜底和 EVAL 的产品需求稿。

不适用于报告、机会判断、复盘、邮件、幻灯片或单纯建议。

## 核心约束

- 输出必须使用固定 PRD 结构。
- 需求必须能被研发、设计、测试、运营理解和验收。
- AI 能力必须写成产品规则和验收要求，不写成模型宣传。
- 同一条规则只在一个主要章节完整出现，避免跨章节重复。
- 禁止输出结构外摘要、亮点总结、版本计划、优先级或 Demo/MVP 口径。
- 飞书交付时，以线上最新文档为准，并保留有价值的图片、画板、表格和资源块。

## 文件结构

`SKILL.md` 是实际生效的 skill 定义文件。

## 本地安装

将仓库中的 `SKILL.md` 放入本地 Codex skill 目录：

```bash
mkdir -p ~/.codex/skills/PRD-writer
cp SKILL.md ~/.codex/skills/PRD-writer/SKILL.md
```

安装后，在 Codex 中通过 `[$PRD-writer]` 调用。

## 维护说明

更新规则时，优先修改 `SKILL.md`。README 只保留用途、安装和维护说明，避免与 skill 正文重复。
